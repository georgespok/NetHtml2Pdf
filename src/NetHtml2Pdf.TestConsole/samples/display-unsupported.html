CSS Display: Unsupported Values - Test Fixture

This fixture demonstrates unsupported CSS display values and their fallback behavior. This tests:
  • Structured warnings for unsupported values
  • Fallback to HTML semantic default
  • Class-based unsupported values
  • Inline style unsupported values
  • Mixed supported and unsupported values
Important: This fixture is designed to test warning behavior and fallback functionality. Unsupported display values should emit structured warnings and fall back to HTML semantic defaults (e.g., div=block, span=inline) without causing errors.

Warning Behavior Expected:
  • Each unsupported display value should emit exactly one structured warning
  • Warnings should include the unsupported value and fallback information
  • Warnings should not be emitted for supported values (block, inline-block, none)
  • Warnings should not be emitted for valid CSS keywords (initial, inherit, unset)
  • Fallback behavior should be consistent and predictable
Overall Expected Results:
  • Unsupported display values should emit structured warnings
  • Elements should fallback to their HTML semantic default (div=block, span=inline)
  • Layout properties should be preserved during fallback
  • Nested elements should follow their own display properties
  • Text content should be preserved
  • Supported values should work normally without warnings
  • Invalid values should be handled gracefully
  • CSS class and inline style unsupported values should both work
